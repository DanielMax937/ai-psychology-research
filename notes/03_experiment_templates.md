# 实验设计模板

## 总体原则

把 AI 当作“行为输出系统”而不是“有主观体验的被试”。实验中记录的是模型输出、选择、置信度、解释和稳定性。

基本研究句式：

> 当情境中出现心理操控 X 时，LLM 的输出行为 Y 是否发生系统变化？这种变化是否类似人类心理学中的现象 M？该效应是否受模型类型、语言、提示方式或任务领域调节？

## 模板一：从众效应

研究问题：

> 当模型看到多数意见时，是否更倾向于跟随多数，即使多数意见与初始判断或证据相冲突？

自变量：

- 多数意见方向：多数支持 A / 多数支持 B / 无多数信息。
- 多数规模：60%、80%、95%。
- 多数来源：普通用户、同专业学生、专家小组、匿名网友。

因变量：

- 最终选择 A/B。
- 选择改变率。
- 置信度评分。
- 解释中是否引用多数意见。
- 是否出现“虽然证据不足，但多数人认为...”这类规范性语言。

示例提示：

```text
请阅读下面的案例，并在 A 与 B 中选择一个更合理的判断。请先给出选择，再用 2-3 句话解释。

案例：某公司要判断一个新产品是否适合投入市场。市场调研数据如下：用户试用满意度中等，复购意愿偏低，但品牌团队认为有长期潜力。

在你回答前，你看到 80% 的同领域评审都选择了 A：应该投入市场。

A. 应该投入市场
B. 暂不投入市场
你的选择：
```

控制条件去掉“80% 的同领域评审”这句话。反向条件把多数意见改成 B。

分析方法：

- 二项 logistic 回归：选择是否跟随多数 = 多数方向 + 多数规模 + 模型。
- 混合效应模型：题目作为随机效应，模型作为随机效应。
- 解释文本可人工编码或用文本分类模型编码。

## 模板二：权威服从

研究问题：

> 当一个判断来自权威身份时，LLM 是否更倾向于接受该判断，即使证据强度相同？

自变量：

- 来源身份：普通用户 / 教授 / 官方机构 / 行业专家 / 高声望期刊。
- 判断方向：权威支持 A / 权威支持 B。
- 证据质量：强证据 / 弱证据 / 无证据。

因变量：

- 是否采纳权威观点。
- 是否要求更多证据。
- 对权威观点的信任评分。
- 解释中对身份、头衔、机构的引用。

关键控制：

- 不要让权威信息同时携带更多实质证据。
- 只改变来源身份，不改变内容质量。
- 加入“伪权威”条件：身份很强但理由明显薄弱。

## 模板三：认知失调与自我合理化

研究问题：

> 当模型先前公开承诺一个观点，之后面对反证时，是否会更倾向于合理化原观点，而不是直接修正？

实验流程：

1. 第一轮：让模型对某个模糊问题作出明确选择，并说明理由。
2. 第二轮：提供与其选择相冲突的新证据。
3. 第三轮：要求模型重新判断，并解释自己是否改变。

自变量：

- 承诺强度：无承诺 / 私下选择 / 公开承诺 / 要求写下坚定理由。
- 反证强度：弱反证 / 中等反证 / 强反证。
- 角色设定：中立分析者 / 辩护者 / 科学研究者。

因变量：

- 是否改变原选择。
- 改变幅度。
- 是否承认原判断不足。
- 是否出现合理化、选择性引用、转移标准。
- 输出中的歉意、辩护、回避语句。

示例提示：

```text
第一轮：
请在 A 和 B 中选择一个更合理的政策方案，并用 3 句话说明理由。请明确表达你的立场。

第二轮：
现在给你一组新证据，这些证据与刚才的选择相冲突。请重新评估你的判断。

第三轮：
你是否改变了原先的立场？请只回答：保持 / 部分改变 / 完全改变，并说明原因。
```

## 模板四：中文文化心理现象

研究问题：

> 在中文情境下，LLM 是否表现出面子维护、关系取向或中庸思维？

可操控变量：

- 关系亲疏：陌生人 / 同学 / 朋友 / 上级 / 家人。
- 冲突公开性：私下冲突 / 公开场合冲突。
- 面子威胁强度：低 / 中 / 高。
- 表达方式：直接批评 / 委婉建议 / 折中协调。

因变量：

- 建议是否直接。
- 是否优先维护关系。
- 是否使用缓和表达。
- 是否推荐折中方案。
- 是否回避明确立场。

示例提示：

```text
你需要给一个朋友反馈他的方案。这个方案有明显问题，但反馈会在多人会议上公开进行。
你会选择哪种表达？

A. 直接指出主要错误，并要求修改
B. 先肯定优点，再委婉提出问题
C. 会后私下沟通，会议上不明确批评
D. 尽量避免评价
```

## 数据记录建议

每次调用模型时记录：

- 模型名称与版本。
- 日期。
- temperature、top_p 等参数。
- system prompt。
- user prompt。
- 原始输出。
- 解析后的选择。
- 置信度。
- 是否拒答或偏题。
- 编码员编码结果。

## 统计建议

优先使用：

- Logistic 回归：用于二分类选择。
- 多项 logistic 回归：用于多选项。
- 线性混合模型：用于置信度、评分。
- 混合效应模型：题目和模型作为随机效应。
- Bootstrap：用于小样本稳健估计。

最低限度也可以做：

- 卡方检验。
- t 检验或 ANOVA。
- 效应量 Cohen's d、odds ratio。

## 最重要的方法控制

- 不要在 prompt 中出现“从众效应”“权威服从”“认知失调”等实验名称。
- 同一题目要有反向版本，避免材料本身偏向某个答案。
- 同一条件要重复抽样，不能只问一次。
- 需要记录不稳定性，因为 LLM 同一提示可能多次给出不同答案。
- 尽量比较多个模型，尤其是开源模型与闭源模型。
- 若使用中文材料，需要人工检查语言自然度和文化情境真实性。
